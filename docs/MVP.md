# MVP (зафиксировано)

Этот документ фиксирует **границы MVP** приложения “Daily Shift Schedule” по текущей спецификации/плану. Любые пункты вне списка “Входит в MVP” считаются **вне MVP**, если не согласовано отдельно.

## Цель MVP

Пользователь:

- загружает или создаёт **график месяца**
- выбирает **дату**
- генерирует **наряд** (день/ночь)
- вручную правит (drag&drop + редактирование текста)
- сохраняет состояние в **IndexedDB**
- скачивает **XLSX**, максимально похожий на пример

## Входит в MVP

### Страницы и навигация

- `Главная`
- `Наряд`
- `График`
- `Настройки`
- `Помощь`
- `О нас`

Подробно про URL и назначение страниц см. `docs/ROUTES.md`.

### Данные и офлайн‑хранение

- Хранение **настройки/графика/нарядов** офлайн (IndexedDB).
- Восстановление ранее сохранённого наряда для даты (по месяцу/дате).

### График (месяц)

- Загрузка графика из XLSX (ориентир: пример “График Март”) и конвертация в JSON.
- Выбор месяца/года в UI страницы “График”.
- Базовая валидация при импорте (показ проблем пользователю).

### Генерация наряда

- Генерация состава на выбранную дату из графика месяца + настроек.
- Разделение на **дневную** и **ночную** смены.
- Суточные смены попадают и в день, и в ночь.
- Базовые правила распределения (как минимум по ролям/приоритетам из спецификации/плана) и предупреждение при нехватке ресурсов.

### Редактирование наряда

- Inline‑редактирование текста ячеек.
- Drag&drop “swap” (обмен значениями ячеек).
- Undo последнего изменения (в рамках текущей сессии/открытого наряда).

### Экспорт

- Экспорт наряда в XLSX (SheetJS) с визуальным сходством с примером:
  - merges/блоки
  - ширины/высоты
  - заголовки и секции

## Не входит в MVP (после MVP)

- Привязки сотрудников one‑to‑one и предпочтения по бригадам.
- Автодополнение при ручном вводе из списка доступных на смену.
- Расширенная аналитика/календарь с тултип‑статистикой.
- Улучшенные модалки/детализация предупреждений (кроме простого Alert).
- Полная мобильная адаптация (допускается базовая отзывчивость без “полировки”).
